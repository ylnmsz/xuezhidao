# 基于大语言模型与文档版面分析技术的智能教辅平台设计与实现

## 摘 要

随着教育信息化建设持续推进，学校教学活动不断向数字化、平台化转型。然而，在基础教育与高等教育场景中，题库建设仍然普遍存在“人工录入成本高、图文公式混排难、数据结构不统一”的问题。尤其在数学、物理等学科中，试题通常包含复杂公式、图片、表格与分段答案解析，传统题库系统难以高质量完成结构化录入，导致教师在题库维护上的时间投入远高于教学创新投入。

针对上述问题，本文设计并实现了一套面向师生协同场景的智能教辅平台“学之道”。系统采用前后端分离架构，前端以 Vue 3 与 Vite 为核心，结合 Pinia、Vue Router 构建响应式交互界面；后端基于 Node.js 与 Express 搭建 RESTful API 服务，使用 PostgreSQL 存储业务数据。平台在功能层面覆盖用户认证、题库管理、作业发布、在线练习、错题本沉淀、消息通知与排行榜等模块，形成“教学任务下发—学生练习反馈—数据回流分析”的闭环。

本文的核心创新在于：将 MinerU 文档版面分析能力与大语言模型结构化抽取能力进行深度融合，构建出一条可工程化落地的 DOCX 智能解析流水线。系统通过文档上传、异步任务轮询、ZIP 结果下载、图片持久化映射、文本分块并发抽取、JSON 纠错与题型归一化等多阶段处理，实现了复杂试卷到结构化题目的自动转换。面对大模型输出不稳定问题，本文提出并实现多层回退的 JSON 解析策略，显著提升了解析成功率与服务鲁棒性。

实验与功能测试结果表明：该系统在多页图文混排试卷场景下能够稳定输出结构化题目，并在并发分块与接口容错机制支撑下保持较好的处理效率。系统在教学辅助、题库建设与学习行为分析方面具有明显实用价值，可为教育信息化平台建设提供可复用的工程实践方案。

关键词：智能教辅；大语言模型；文档版面分析；题库自动构建；Vue 3；Node.js

## Abstract

With the continuous advancement of educational informatization, teaching activities are increasingly shifting toward digital and platform-based models. However, in both K-12 and higher education scenarios, question bank construction still faces persistent challenges, including high manual input cost, difficulties in handling mixed text-image-formula content, and inconsistent data structures. In mathematics and physics subjects especially, exam papers often contain complex formulas, diagrams, and segmented explanations, making high-quality structured entry difficult for traditional systems.

To address these issues, this paper designs and implements an intelligent educational assistant platform named XueZhiDao. The system adopts a front-end/back-end separated architecture. The front-end is built with Vue 3 and Vite, combined with Pinia and Vue Router for responsive interaction. The back-end is implemented with Node.js and Express to provide RESTful APIs, while PostgreSQL is used for persistent data storage. Functionally, the platform supports authentication, question bank management, assignment distribution, online practice, error-book accumulation, messaging, notifications, and rankings, forming a closed loop from teaching delivery to learning feedback and analytics.

The key contribution of this work is the deep integration of MinerU document layout analysis and LLM-based structured extraction, forming an engineering-ready DOCX intelligent parsing pipeline. Through multi-stage processing including file upload, asynchronous polling, ZIP download, image persistence and mapping, text chunking with concurrency, JSON repair, and question normalization, the platform achieves automated conversion from complex papers to structured question entities. To address unstable model outputs, this paper proposes and implements a multi-layer fallback JSON parsing strategy, significantly improving parsing success rate and system robustness.

Experimental and functional evaluations show that the system can stably generate structured questions from multi-page mixed-layout documents while maintaining good processing efficiency through concurrent chunking and fault-tolerant strategies. The platform demonstrates strong practical value for educational assistance, intelligent question bank construction, and learning behavior analysis.

Keywords: intelligent tutoring platform, large language model, document layout analysis, automated question bank construction, Vue 3, Node.js

## 1 绪论

### 1.1 研究背景

近年来，国家层面持续推动教育数字化战略，在线教学、混合教学、精准教学逐渐从“补充手段”演变为“基础能力”。在此背景下，题库作为教学资源数字化的核心载体，其质量与规模直接影响作业布置效率、测评公平性和个性化学习能力。

在传统流程中，教师通常需要手动完成以下任务：

1. 将纸质或 Word 试卷逐题拆分。
2. 将题干、选项、答案、解析分别录入系统。
3. 对公式、上下标、特殊符号进行额外排版。
4. 将配图单独上传并与题目字段绑定。

该流程操作重复度高、容错率低。一旦试卷结构复杂或格式不规范，人工处理耗时会显著增加，导致教师在机械性工作中消耗大量时间，不利于教学设计与课堂创新。

与此同时，生成式人工智能和文档智能解析技术正在快速发展。大语言模型在语义理解和信息抽取上表现突出，文档版面分析模型在图文分离与结构重建方面持续突破。将二者结合并工程化应用于题库建设，具备明确的现实价值和研究意义。

### 1.2 研究目的与意义

本课题的研究目标是设计并实现一套“可落地、可维护、可扩展”的智能教辅平台，重点解决复杂试卷结构化录入问题。具体意义包括：

1. 提升教师工作效率：将题库录入由“逐题手工输入”转变为“文档批量自动抽取”。
2. 增强平台资源生产能力：快速沉淀高质量题目数据，为作业、练习、测评提供基础。
3. 提高学生学习反馈质量：通过错题本、排行榜、学习画像等模块形成数据驱动的学习支持。
4. 形成可复用工程方案：提供一条从文档解析到题库入库的通用技术路径，可扩展到更多教育场景。

### 1.3 国内外研究现状

当前题库系统研究主要集中在三个方向：

1. 业务流程型题库平台：强调课程管理、作业发布、成绩统计，但在复杂文档自动结构化方面能力有限。
2. OCR 文本识别型系统：能够识别图片中的文字，但对版面结构理解不足，难以可靠地恢复题目边界与图文关系。
3. 大模型抽取型方案：语义抽取能力强，但稳定性与可控性不足，常见问题包括输出格式漂移、JSON 不合法、字段缺失等。

因此，单一技术路线难以满足“复杂文档高质量结构化”的工程要求。本文采用“MinerU + LLM + 工程化容错”的组合方案，通过分阶段治理数据质量，实现兼顾准确性与可用性的系统落地。

### 1.4 研究内容

围绕上述问题，本文完成如下工作：

1. 设计并实现基于 Vue 3 + Node.js + PostgreSQL 的前后端分离系统架构。
2. 完成用户、班级、作业、提交、题库、通知、消息、错题本等核心业务模块开发。
3. 构建 DOCX 智能解析流水线，打通上传、解析、清洗、入库全过程。
4. 实现多层 JSON 解析容错机制，解决 LaTeX 反斜杠与控制字符导致的结构化失败问题。
5. 完成系统功能测试与性能观测，验证方案可行性与稳定性。

## 2 相关技术与理论基础

### 2.1 前端技术体系

#### 2.1.1 Vue 3 与 Composition API

Vue 3 采用响应式依赖追踪机制，具备良好的组件化能力与开发体验。Composition API 使状态逻辑按功能聚合，较 Options API 更利于复杂页面维护。本文在学生端练习页、题库预览页、个人中心等页面中大量使用组合式写法，使业务逻辑和视图渲染解耦。

#### 2.1.2 Vue Router

系统路由采用前端路由机制，支持登录页、教师端、学生端、消息中心、题库管理等多页面切换。通过路由守卫结合本地 token 状态，实现基础权限控制与页面访问约束，避免未登录状态访问核心业务页面。

#### 2.1.3 Pinia 状态管理

系统使用 Pinia 管理跨组件共享状态，典型包括用户信息、权限角色、当前班级上下文等。其优势在于类型友好、模块粒度清晰、调试便利，适合中等复杂度业务系统。

#### 2.1.4 KaTeX 公式渲染

数学试题中普遍包含 LaTeX 公式。系统在前端引入 KaTeX 对题干、选项、答案解析进行渲染，保证积分、矩阵、根式、分段函数等表达式显示规范，提升学生阅读体验与答题准确性。

### 2.2 后端技术体系

#### 2.2.1 Node.js 与 Express

Node.js 的事件循环与异步 I/O 模型适合高并发 Web 服务场景。Express 结构简洁、生态成熟，便于快速搭建控制器、路由、中间件。本项目将控制层、路由层、服务层拆分，提升了代码可维护性。

#### 2.2.2 PostgreSQL 数据库

PostgreSQL 具有事务可靠、SQL 能力强、类型系统丰富等特性。系统采用 UUID 作为主键，降低分布式场景冲突概率；关键业务表通过外键约束保证数据一致性，支持后续扩展统计分析与报表查询。

#### 2.2.3 JWT 与密码安全

登录态采用 JWT 方案。用户密码使用 bcryptjs 加盐哈希存储，避免明文泄露风险。前端请求统一附加 Authorization 头，后端在鉴权中间件校验 token 合法性，实现无状态身份识别。

### 2.3 文档解析与大模型技术

#### 2.3.1 MinerU 版面解析

MinerU 接收 DOCX 后返回结构化提取结果，包括 markdown 文本与图像资源压缩包。该过程可保留题目逻辑顺序，识别插图位置，为后续语义抽取提供可靠输入。

#### 2.3.2 LLM 结构化抽取

系统将解析后的文本输入大模型，通过严格 Prompt 约束输出字段，要求返回 JSON 数组，字段包含题型、题干、选项、答案、解析、学科、年级、难度等。相比规则匹配，LLM 在语义理解与上下文补全方面优势明显。

#### 2.3.3 工程化容错理论

在生产场景中，“模型输出不稳定”是常态，而非异常。系统采用多阶段回退策略：

1. 直接 JSON.parse。
2. 去围栏、截取数组候选再次解析。
3. 对字符串换行、控制字符、非法反斜杠进行修复。
4. 提取首个对象并兜底组装数组。

通过“先快后稳、逐级回退”的工程思想，显著提高系统成功率。

## 3 需求分析与可行性分析

### 3.1 业务场景分析

项目主要服务两个核心角色：教师与学生。

教师侧关注“资源生产效率”和“教学管理效率”；学生侧关注“作答体验”和“学习反馈质量”。因此系统需要同时满足管理型需求与学习型需求。

### 3.2 功能需求分析

#### 3.2.1 用户与认证模块

1. 用户注册、登录、资料维护。
2. 不同角色视图入口与基础权限控制。
3. token 过期自动失效处理与重登录引导。

#### 3.2.2 题库管理模块

1. 题库列表查询、筛选、详情查看。
2. 手工新增题目与编辑。
3. CSV 批量导入。
4. DOCX 批量智能解析导入。

#### 3.2.3 作业与提交模块

1. 教师创建作业并关联班级。
2. 学生提交作业答案或内容。
3. 教师查看提交记录并评分。

#### 3.2.4 错题本与学习分析模块

1. 自动记录学生错题。
2. 支持错题复习与备注。
3. 生成学习统计与排行榜展示。

#### 3.2.5 消息与通知模块

1. 系统消息发送与查看。
2. 关键事件通知（作业发布、截止提醒等）。

### 3.3 非功能需求分析

#### 3.3.1 性能需求

1. 常规接口响应时间应控制在可交互范围内。
2. 文档解析属于重任务，应支持异步处理与状态查询。
3. 对长文本抽取需通过分块与并发降低总耗时。

#### 3.3.2 可靠性需求

1. 支持异常输入与第三方接口失败时的错误提示。
2. 关键流程需具备可恢复能力，如解析失败可重试。

#### 3.3.3 安全性需求

1. 用户密码加密存储。
2. 受保护接口必须鉴权。
3. 文件上传限制类型与大小，防止恶意文件注入。

#### 3.3.4 可维护性需求

1. 前后端分层清晰。
2. 控制器与服务逻辑解耦。
3. 关键算法函数化，便于调优和替换。

### 3.4 可行性分析

#### 3.4.1 技术可行性

本项目依赖的主流技术栈生态成熟，社区资源丰富。Vue 3、Express、PostgreSQL 均具备较强工程可落地性；MinerU 与 LLM 通过标准 HTTP 接口对接，集成成本可控。

#### 3.4.2 经济可行性

系统开发使用开源框架，部署可在普通云主机完成。模型调用成本与文档解析成本可通过分块策略和参数优化进行控制，具备教育场景中的可持续运行基础。

#### 3.4.3 操作可行性

前端交互面向教师与学生常见使用习惯设计，页面逻辑清晰。题库导入流程已封装为“上传-等待-预览-入库”的可理解操作链，学习成本较低。

## 4 系统总体设计

### 4.1 系统架构设计

系统采用 B/S 架构，分为展示层、业务层、数据层、外部能力层：

1. 展示层：Vue 前端应用，负责页面渲染与用户交互。
2. 业务层：Express API 服务，负责业务规则执行。
3. 数据层：PostgreSQL，负责持久化存储。
4. 外部能力层：MinerU + LLM，负责文档解析与语义抽取。

数据流主路径如下：

1. 用户在前端发起请求。
2. 后端路由分发到控制器。
3. 控制器调用服务逻辑与数据库。
4. 若涉及文档解析，则调用外部能力层。
5. 返回结构化结果并在前端展示。

### 4.2 模块设计

#### 4.2.1 控制器层设计

后端控制器按业务域划分，包含认证、用户、班级、作业、提交、题库、消息、通知、错题本、排行榜等。该设计使单一控制器职责明确，便于测试与维护。

#### 4.2.2 服务层设计

服务层封装复杂业务流程，例如 token 签发、DOCX 解析、题目归一化等。控制器仅负责请求参数处理与响应返回，降低耦合。

#### 4.2.3 路由与中间件设计

路由层按照 REST 风格组织，搭配鉴权中间件、上传中间件进行统一入口控制。文件上传采用 multer 内存存储，限制上传类型与体积，减少非法输入。

### 4.3 数据库设计

根据 `schema.sql`，系统核心数据表包括：

1. `users`：用户基础信息与学习统计字段。
2. `classes`：班级信息。
3. `class_members`：班级成员关系。
4. `assignments`：作业任务信息。
5. `submissions`：学生提交与评分记录。
6. `question_banks`：题库元信息。
7. `questions`：题目主体数据。
8. `messages`：消息通信记录。
9. `notifications`：通知记录。
10. `error_book`：错题本记录。

#### 4.3.1 关键字段说明

1. 用户表中 `level`、`streak_days`、`weekly_study_hours`、`accuracy` 为学习画像与激励机制预留。
2. 题目表中 `stem_image_url`、`option_images` 解决图文混排题存储问题。
3. 作业与提交表通过外键关联形成“任务-提交-评分”链路。

#### 4.3.2 关系设计说明

1. 一个班级可对应多个成员与多条作业。
2. 一个作业可对应多个提交记录。
3. 一个题库可包含多个题目。
4. 一个用户可拥有多条错题记录。

### 4.4 接口设计原则

1. URL 语义化，便于理解与调用。
2. 请求与响应统一 JSON 结构。
3. 错误码与错误信息清晰可追踪。
4. 重要接口进行鉴权校验。

### 4.5 安全设计

1. 密码哈希存储，避免明文泄漏。
2. 登录凭证由 JWT 提供，后端验证签名和时效。
3. 前端在 401 时自动清理无效凭证，降低脏状态风险。
4. 上传类型白名单限制：CSV、DOCX。

## 5 系统详细实现

### 5.1 用户认证模块实现

#### 5.1.1 注册流程

注册接口校验邮箱、密码、姓名等必填字段，并设置密码长度下限。后端查询邮箱唯一性后，使用 bcrypt 哈希密码，写入用户表并返回 token 与用户信息。

#### 5.1.2 登录流程

登录接口根据邮箱查询用户并校验密码哈希。校验通过后签发 token，返回前端保存。前端后续请求通过 Authorization 头携带该 token。

#### 5.1.3 前端鉴权处理

前端 API 封装统一读取本地 token，自动附加请求头。当后端返回 401 时，清理本地缓存并重定向登录页，避免用户在失效状态下重复触发失败请求。

### 5.2 题库模块实现

#### 5.2.1 列表与筛选

题库列表接口支持按学科、难度、题库 ID 等参数过滤，后端动态拼接条件查询，返回题目集合供前端展示。

#### 5.2.2 CSV 导入

系统支持通过 CSV 批量导入题目。上传中间件先做文件类型校验，再由解析逻辑将行数据映射到题目字段，完成批量入库。

#### 5.2.3 DOCX 批量解析导入

该模块是系统关键创新实现，处理流程如下：

1. 接收前端上传的 DOCX 文件。
2. 将文件写入服务器上传目录并生成唯一文件名。
3. 调用 MinerU 上传申请接口获取 `batch_id` 与预签名上传地址。
4. 使用 PUT 上传文件内容到 MinerU 存储。
5. 轮询结果接口直到任务完成或失败。
6. 下载结果 ZIP 文件并解压。
7. 读取 `full.md`，并将 ZIP 中图片复制到平台静态目录。
8. 将 markdown 图片链接映射为平台可访问 URL，并替换成 `[img:URL]` 占位。
9. 对长文本执行分块并发调用 LLM，获取结构化题目。
10. 对输出进行多层 JSON 修复与字段归一化。
11. 返回前端预览并支持入库。

### 5.3 文档解析关键算法实现

### 5.3.1 文本分段策略

函数 `splitSections` 通过匹配中文大题编号（如“一、二、三”）提取章节边界。`chunkTextBySections` 在尽量保持语义完整的前提下将文本切分为若干块，避免模型上下文超限。

### 5.3.2 并发执行策略

函数 `mapWithConcurrency` 通过限制 worker 数实现受控并发。相比串行调用，大幅降低总耗时；相比无限并发，可避免模型接口拥塞与失败率升高。

### 5.3.3 图片占位与映射策略

markdown 中的图片通过正则替换为 `[img:URL]` 占位；后续在题目归一化阶段，根据 stem 与 option 内容中的占位信息将图片归入 `stemImageUrl` 或 `optionImages`，实现图文正确关联。

### 5.3.4 JSON 容错解析策略

系统实现 `tryParseJson` 多轮回退逻辑，核心步骤包括：

1. 去除代码围栏与噪声文本。
2. 提取最可能的 JSON 数组片段。
3. 修复字符串内部换行。
4. 修复非法反斜杠和控制字符。
5. 必要时提取首个对象并包装为数组。

该策略有效应对 LLM 输出中的常见异常，如：

1. 末尾多余逗号。
2. 未转义反斜杠。
3. 包含不可见控制符。
4. 非 JSON 文本混入。

### 5.4 题目归一化实现

`normalizeItemImages` 针对模型输出不一致问题做了多项标准化处理：

1. 统一题型字段。
2. 自动补全空选项图数组。
3. 从题干中拆分“答案/解析”片段。
4. 清除文本中占位符并保留图片 URL。
5. 对缺失字段应用兜底默认值。

### 5.5 前端页面实现说明

#### 5.5.1 登录与注册页

登录页负责用户认证与错误提示。去除无关身份切换后，交互路径更清晰。登录成功后保存 token 与用户信息并跳转工作台。

#### 5.5.2 题库管理页

题库页支持题目列表展示、筛选、导入和预览。针对解析出的图文题，前端通过分段渲染文本和图片片段，结合 KaTeX 对公式进行实时展示。

#### 5.5.3 学生练习页

练习页按题型组件化渲染，支持单选、多选、判断、填空、简答。提交后可进入错题沉淀流程并参与学习统计。

### 5.6 消息、通知与排行榜实现

#### 6.4.1 文档解析耗时观测

#### 6.4.2 并发稳定性观测

在适中并发限制下，模型调用失败率可控；并发过高时可能因上游限流导致重试上升。系统当前采用受控并发值作为折中方案。

1. 未鉴权访问受保护接口被拒绝。
2. 非法文件类型上传被拦截。
3. 弱口令与缺失字段在注册环节得到校验。

系统在功能完整性、性能可接受性与容错能力方面满足毕业设计预期目标。特别是在复杂文档自动结构化环节，已具备可实际使用的工程价值。

## 7 总结与展望

### 7.1 工作总结

本文围绕“复杂试卷自动结构化录入”这一核心问题，完成了智能教辅平台“学之道”的设计与实现。系统构建了从文档上传到题目入库的完整链路，并通过前后端模块化设计保障了业务扩展性。实践表明：

1. 文档版面分析与大模型结合在教育题库场景中具有较高可行性。
2. 仅依赖模型输出难以保证稳定，必须引入工程化容错机制。
3. 分块并发策略能显著优化解析耗时，是性能提升关键。

### 7.2 创新点总结

1. 提出并实现“MinerU + LLM + 多层容错”一体化解析流水线。
2. 实现针对 LaTeX 与非标准 JSON 的后端回退解析策略。
3. 在真实教学业务流程中完成系统级落地，形成师生协同闭环。

### 7.3 不足分析

1. 对极端复杂版式文档仍可能需要人工校对。
2. 当前解析质量受第三方模型能力波动影响。
3. 缺少更大规模真实学校数据下的长期对比评估。

### 7.4 未来工作展望

1. 引入任务队列与重试机制，提高高峰期吞吐与稳定性。
2. 增加解析结果可视化校对工具，降低人工二次编辑成本。
3. 扩展更多题型元数据（知识点、能力维度、认知层级）。
4. 研究模型微调或本地化部署方案，降低外部依赖与成本。
5. 引入更完善的自动化测试体系和监控告警机制。
