# 🎓 学之道 (XueZhiDao) - 智能教辅与题库管理平台

## 📝 1. 项目简介

**“学之道”** 是一个基于前后端分离架构构建的现代化智能教育与教辅管理平台。该平台为教师和学生提供了双端的专属工作台。其最大特色在于深度融合了**人工智能（LLM）**与**高级文档解析技术（MinerU）**，实现了将排版复杂的试卷文档（如 DOCX）一键自动化解析为包含公式、插图及答案解析的结构化题库，极大减轻了教师录入试题的负担。

同时，平台结合了学生练题、错题本收录、智能学情分析统计、作业分发与 AI 在线答疑等功能，打造了一个闭环的数字化教育生态系统。

## 🛠️ 2. 技术栈架构

项目采用了目前前端主流、后端轻量高效的全面现代化技术栈：

### 前端技术 (Frontend)

- **核心框架**: Vue 3 (Composition API)
- **路由状态**: Vue Router 4 + Pinia
- **构建工具**: Vite 8 (极致的冷启动及热更新速度)
- **数学公式渲染**: KaTeX 引擎 (用于渲染题干和选项中的高数 LaTeX 公式)
- **UI & 样式**: 响应式布局 + 现代化组件编排，并集成 Cropper.js 实现头像剪裁等交互。

### 后端技术 (Server)

- **运行环境**: Node.js (v20+)
- **Web 框架**: Express.js (提供标准的 RESTful API)
- **数据库**: PostgreSQL (`pg` 驱动)，配合结构化的 SQL (Schema)
- **身份认证**: JWT (JSON Web Token) 无状态鉴权及 Bcrypt 密码加密
- **文件处理**: Multer (文件上传)，Adm-zip (解压 MinerU 解析出的富文本素材资源)

### 人工智能与解析服务库仓库 (AI & NLP)

- **MinerU/OCR引擎**: 将非结构化的 DOCX/PDF 文档进行版面分析，提取为带图片坑位 (`[img:URL]`) 的 Markdown 文本，并分离图片资源。
- **大语言模型 (OpenAI API 兼容)**: 编写复杂 Prompt，对长文本进行分块 (Chunking) 处理，从非结构化文本中精准抽取为特定的 JSON 题库结构 (题干、选项、答案、解析、难度)。

## 🚀 3. 核心功能模块

### 👤 3.1 用户角色与鉴权中心体系

- **双端视图**: 学生端 (Student Dashboard) 和教师端 (Teacher Dashboard) 具备完全独立的视图和业务逻辑。
- **无感鉴权**: 登录成功后发放 JWT Token 存储本地，拦截失效状态（统一 401 拦截），具备防刷白注册和个人资料/头像管理功能。

### 📚 3.2 智能题库管理与自动录入 (项目核心亮点)

- **复杂文档智能解析流水线 (Pipeline)**: 建立了一套 `[文档上传] -> [MinerU接口提取Markdown及ZIP图包] -> [解压匹配图像占位符] -> [大模型并发切割解析成JSON数组] -> [异常JSON容错格式化] -> [入库]` 的全自动题库录入流程。
- **鲁棒性解析算法**: 后端对复杂公式转义字符具有极强的容错能力。如实现了 `bruteForceEscapeBackslashes` 兜底算法，可完美化解大模型回答中由于 LaTeX 公式反斜杠（如 `\left|` 等）缺失导致的 JSON 解析崩溃（`SyntaxError`）问题。
- **题型全覆盖**: 支持解析单选题、多选题、判断题、填空题、简答题，并自动拆分题干、选项配图和答案解析。

### 📝 3.3 作业与考试测评系统

- **题库组卷与分发**: 教师可在 `AssignHomework` 发发作业并管理班级。
- **沉浸式练习 (Immersive Practice)**: 为学生提供无干扰的答题环境，包含单选、多选、判断等独立渲染模版，支持多媒体(图片/公式)渲染。

### 📊 3.4 动态学情分析与图表 (Analytics & Grading)

- **可视化仪表盘**: 展示班级作业完成率、正确率等分析数据。
- **排行系统 (Rankings)**: 学生/教师专属的排行榜体系，建立学习激励机制。

### 📖 3.5 智能错题本 (Error Book)

- 自动将答错的题目归档入 `StudentErrorBook`。
- 学生可对错题进行重练，系统可针对错题记录计算薄弱知识点（Subject/Grade 跟踪）。

### 🤖 3.6 AI 助教助手 (Ai Assistant)

- 独立的高级 AI 智能对话界面，可针对题目给学生提供答疑、思路引导而不直接给出答案操作。

## 🎯 4. 毕业设计技术难点与创新点

1. **大规模文件上下文识别 (大模型局限性突破)**：
   为避免大模型一次性处理万字试卷引发长度截断（Token Limit）、响应过慢和幻觉报错，项目设计了 `chunkTextBySections` 分段切割算法，并采用 `mapWithConcurrency` 在 Node.js 中实现**批量多线程异步请求** API 抽取内容，将长试卷解析时间从按分钟算降低至秒级。
2. **多模态图文混排题目的对位还原**：
   从原题提取的选项 A/B/C/D 中可能含有排版不规则的图片。通过与 MinerU 处理完后反馈的包建立映射表（`imageMap`），配合后端将占位符 `[img:URL]` 替换为服务端静态资源路径的渲染机制，成功攻克了“文字+公式+图片”混排考题提取这一业界难题。
3. **高容错的后端中间件与业务设计**：
   在请求链路中深度处理异常拦截。例如请求未携带 Token 或过期（如 401 状态错误），前端网络层封装了自动强校验清洗缓存和重定向等操作，提升了系统鲁棒性和用户体验。
