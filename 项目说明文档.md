# 学之道 (XueZhiDao) 项目说明文档

## 1. 项目简介

“学之道”是一个基于前后端分离架构的智能教辅与题库管理平台。系统面向教师和学生提供双端工作台，覆盖题库建设、班级管理、作业发布、学生练习、错题沉淀、排行榜、学情分析与 AI 辅助答疑等教学闭环。

项目的核心能力是将教师已有的试卷、练习册和 DOCX 文档转化为可入库、可练习、可统计的结构化题目。后端通过 MinerU 文档解析、OpenAI API 兼容模型抽取、JSON 容错修复、图片资源映射和题型归一化，将非结构化材料转换为题库数据，减少教师手工录题成本。

## 2. 技术栈

### 2.1 前端

- Vue 3：页面与组件开发。
- Vue Router：页面路由和角色权限守卫。
- Pinia：用户状态管理。
- Vite 8：开发服务器与生产构建。
- KaTeX：数学公式渲染。
- Cropper.js：头像裁剪交互。

### 2.2 后端

- Node.js 20+：服务端运行环境。
- Express：REST API 服务。
- PostgreSQL：业务数据存储。
- JWT：无状态登录鉴权。
- bcryptjs：密码哈希。
- Multer：文件上传。
- adm-zip：解析服务返回资源包处理。

### 2.3 外部能力

- MinerU：文档版面解析、Markdown 提取和图片资源分离。
- OpenAI API 兼容模型：试题结构化抽取、答案和解析提取。

## 3. 核心功能

### 3.1 用户、登录与权限

- 支持教师和学生两类角色。
- 登录成功后签发 JWT，前端统一保存登录态。
- 前端路由根据 `meta.role` 限制教师端和学生端页面访问。
- API 请求统一附带 Authorization，401 时清理本地登录态并跳转登录页。
- 支持个人资料、头像、学习目标和提醒时间维护。

### 3.2 教师端

- 教师工作台：展示班级、学生、作业和学情概览。
- 班级管理：创建班级、查看成员、生成邀请码。
- 题库管理：创建题库、筛选题目、手动录入题目、批量解析题目。
- 文档解析：支持 DOCX 上传、MinerU 解析、LLM 抽取、图片占位符替换和结构化入库。
- 作业发布：从题库选择题目，设置班级、标题、说明和截止时间。
- 作业管理与批改：查看提交情况，对主观题和作业结果进行批改。
- 学情分析：按班级和学生统计完成率、正确率、薄弱点等数据。
- 教师排行榜：展示班级或教学相关排名数据。

### 3.3 学生端

- 学生工作台：展示学习概览、练习进度、作业状态和排名信息。
- 加入班级：通过教师生成的邀请码加入班级。
- 作业列表：查看待完成和已完成作业。
- 沉浸式练习：支持单选、多选、判断等题型的集中作答。
- 练习会话：按题目集合进行练习，提交答案后记录结果。
- 错题本：自动收录错题，支持查看解析和重新练习。
- 学情分析：展示正确率、练习量、学习时长和薄弱学科。
- 学生排行榜：基于积分、战力等指标展示学习排名。
- AI 助手：提供面向学生的答疑和思路引导。

### 3.4 题型与内容渲染

当前项目支持的题型包括：

- 单选题
- 多选题
- 判断题
- 填空题
- 简答题
- 阅读理解
- 连线题
- 翻译题
- 改错题
- 编程题
- 完形填空
- 古诗默写
- 拼音填空
- 其他扩展题型

前端通过专用上传表单和题型渲染组件处理不同题型，数学公式由 `src/utils/renderMath.js` 结合 KaTeX 渲染。

## 4. 智能题库解析流程

```text
上传 DOCX/文本
-> 后端接收文件
-> MinerU 提取 Markdown 和图片资源
-> 解压资源包并建立图片映射
-> 文本分段与题型识别
-> LLM 抽取结构化题目 JSON
-> JSON 容错修复与字段归一化
-> 写入 question_banks / questions
-> 前端题库页面展示和编辑
```

关键实现点：

- `server/src/services/convertDoc.js`：处理 DOCX 与解析服务相关流程。
- `server/src/services/docxQuestionParser.js`：负责题目抽取、分块、容错和归一化。
- `src/components/questions/upload/`：不同题型的前端录入组件。
- `src/utils/renderMath.js`：对公式和安全标签进行渲染处理。

## 5. 数据库模块

主要数据表包括：

- `users`：用户、角色、个人资料、积分、战力、练习统计。
- `classes`：班级信息。
- `class_members`：班级成员关系。
- `class_invites`：班级邀请码。
- `assignments`：作业信息和题目集合。
- `submissions`：作业提交与批改记录。
- `question_banks`：题库分组。
- `questions`：题目主体、答案、解析、题型和学科。
- `messages`：站内消息。
- `notifications`：通知。
- `error_book`：学生错题记录。
- `points_log`：积分变动记录。

完整字段说明见 `server/DATABASE_SCHEMA.md`，初始化和迁移脚本见 `server/sql/schema.sql`。

## 6. API 模块

后端统一挂载在 `/api` 下，主要模块包括：

- `/api/auth`：注册、登录。
- `/api/users`：用户资料、头像、密码、学习设置。
- `/api/classes`：班级管理。
- `/api/class-members`：班级成员管理。
- `/api/invites`：班级邀请码。
- `/api/question-banks`：题库分组。
- `/api/questions`：题目管理和解析入库。
- `/api/assignments`：作业发布和查询。
- `/api/submissions`：作业提交和批改。
- `/api/practice`：练习记录和结果提交。
- `/api/errorbook`：错题本。
- `/api/analytics`：学情分析。
- `/api/rankings`：排行榜。
- `/api/messages`：站内消息。
- `/api/notifications`：通知。
- `/api/uploads`：上传资源访问。

## 7. 本地运行

### 7.1 环境要求

- Node.js 20.19+ 或 22.12+
- PostgreSQL
- npm

### 7.2 安装依赖

```bash
npm install
cd server
npm install
```

### 7.3 后端环境变量

在 `server/.env` 中配置：

```env
PORT=4000
DATABASE_URL=postgres://用户名:密码@localhost:5432/数据库名
JWT_SECRET=请替换为安全随机字符串
JWT_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173

OPENAI_API_KEY=
OPENAI_MODEL=deepseek-chat
OPENAI_BASE_URL=https://api.openai.com/v1

MINERU_API_URL=https://mineru.net/api/v4/file-urls/batch
MINERU_API_TOKEN=
MINERU_API_RESULT_URL=https://mineru.net/api/v4/extract-results/batch
MINERU_MODEL_VERSION=vlm
```

### 7.4 初始化数据库

```bash
cd server
npm run migrate
```

### 7.5 启动服务

后端：

```bash
cd server
npm run dev
```

前端：

```bash
npm run dev
```

默认访问地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:4000/api`

## 8. 常用命令

npm run build
